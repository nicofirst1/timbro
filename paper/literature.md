## Paper summaries

### zhou2026 — Externalization in LLM Agents: A Unified Review of Memory, Skills, Protocols and Harness Engineering

Systems-level review framing LLM agent externalization across three dimensions — Memory, Skills, Protocols — as a Norman-style cognitive-artifact transformation. The Skills section (§4) covers specification, discovery, progressive disclosure, execution binding, and composition, plus four acquisition pathways (authored, distilled, discovered, composed). Purely architectural: no linguistic/stylistic measurement of skill documentation, no adoption or execution outcomes. Orthogonal to our contribution — treats skills as architectural objects, we treat skill files as linguistic documents with measurable features tied to adoption/execution. Good one-line intro citation for "why skills matter" framing; nothing to reuse methodologically.
→ full notes: literature_full.md#zhou2026

### liang2026-skillnet — SkillNet: Create, Evaluate, and Connect AI Skills

Proposes a three-layer skill ontology (taxonomy, relation graph, package library), an automated multi-source skill-creation pipeline, and a 5-dimension evaluation protocol (Safety, Completeness, Executability, Maintainability, Cost-awareness) validated with near-perfect human/LLM inter-rater agreement (QWK ~1.0). Curates 200K+ skills across multiple ecosystems; shows +40% reward / -30% interaction steps on ALFWorld/WebShop/ScienceWorld versus ReAct/Expel/Few-Shot baselines. Compares six marketplaces (ClawHub, SkillsMP, SkillHub, Skills.sh, SkillNet, others). No linguistic feature analysis, no prose-style variation. RQ3 gains a validated Executability-evaluation precedent; RQ1 gains nothing (content/quality, not language).
→ full notes: literature_full.md#liang2026-skillnet

### li2026-agentskillos — Organizing, Orchestrating, and Benchmarking Agent Skills at Ecosystem Scale

Shanghai AI Lab paper proposing *AgentSkillOS*: (1) **Manage Skills** — organizes a large skill ecosystem into a *capability tree* via recursive LLM-driven categorization, so retrieval can go coarse-to-fine instead of flat semantic search over the whole pool; (2) **Solve Tasks** — retrieves a relevant skill subset from the tree, composes them into a DAG (three orchestration strategies: quality-first, efficiency-first, simplicity-first), then executes the DAG with dependency-aware layering. Evaluated on a 30-task, 5-category benchmark (data computation, document creation, motion video, visual design, web interaction) using **LLM-based pairwise judging in both orderings + a Bradley–Terry model** to get bias-mitigated, fine-grained ranking scores — tested across three ecosystem scales (200/1K/200K skills). Key finding: DAG-based orchestration substantially outperforms flat skill invocation even with the *identical* oracle skill set — structured composition, not mere availability, is what unlocks skill value. Cites Ling et al. (2602.08004) for the "280,000+ skills publicly available" ecosystem-scale framing, and cites SkillNet as related organizing work.

Relevance to us: no linguistic feature analysis, no adoption-prediction, no RQ1 overlap. But their **Bradley–Terry pairwise-judging protocol is a strong candidate evaluation method for the RQ3 pilot** if we want continuous quality/success scores rather than binary pass/fail — position-bias mitigation (judging both orderings) is a detail worth adopting directly. Their scale experiment (200 → 200K skills) is also a useful precedent for how retrieval/orchestration degrades as a skill library grows, tangential to but consistent with our corpus-scale concerns.
→ full notes: literature_full.md#li2026-agentskillos

### hu2026-clawhub — Red Skills or Blue Skills? A Dive Into Skills Published on ClawHub

Analyzes 26,502 ClawHub skills via a three-stage crawl+normalize pipeline: functional clustering (TF-IDF + k-means, 10 clusters/language), download-distribution analysis (log-normal, most skills 30–60 downloads, <10% exceed 1,000), and risk classification (12 classifiers, best Logistic Regression 72.62% acc / 78.95% AUROC, documentation completeness as top feature). Key finding for us: **English skills cluster around infrastructure/tooling (Developer Tools 36.3%), Chinese skills around application scenarios (media, social, finance)** — the closest existing empirical hint that ecosystems/languages produce distinct "dialects," though measured via bag-of-words topic clustering, not linguistic features. Their download distribution is a useful adoption-proxy baseline for RQ2. Corpus/crawl methodology (identity/metadata/content/risk/effectiveness normalization) is transferable to our own corpus build.
→ full notes: literature_full.md#hu2026-clawhub

### liu2026-skillsvote — SkillsVote: Lifecycle Governance of Agent Skills from Collection, Recommendation to Evolution

MemTensor paper treating agent skills as *lifecycle artifacts*: profiles a **million-scale open-source skill corpus** for environment requirements, quality, and verifiability; before execution, runs agentic library search (not static semantic matching) over structured skill folders to expose only relevant instructional context; after execution, decomposes trajectories into skill-linked **subtasks** (the smallest unit with one objective, one evaluation signal, at most one associated skill) and attributes outcomes to skill-guided execution vs. independent exploration vs. environment; only successful, reusable subtask evidence is allowed to trigger an "evidence-gated" skill-library update (edit existing skill / create new skill / skip). Evaluated on Terminal-Bench 2.0 and SWE-Bench Pro across three Codex model/effort configs (GPT-5.2 medium, GPT-5.4 mini medium, GPT-5.5 xhigh): improves avg@5 accuracy on Terminal-Bench 2.0 by +0.9–2.6pp and avg@1 resolve rate on SWE-Bench Pro by +1.2–2.7pp over no-skills baselines, beating ReasoningBank and a naive skill-creator baseline on most settings. Also shows pre-task recommendation reduces *negative* transfer from irrelevant/weakly-related skills (mean task-level contribution goes from +3.3/-6.7 without recommendation to a balanced +6.0/-6.0 with it).

Relevance to us: no linguistic feature analysis, no instruction-dialect framing — content/quality and lifecycle governance, not prose style. But two things are directly reusable: (1) their **subtask-level attribution** design (one objective / one evaluation signal / at most one skill per unit) is a clean template for isolating what to credit when RQ3 measures whether a restyled skill's success differs from the original's; (2) their finding that **naive/negative skill exposure actively hurts task success** (not just "no effect") is a concrete reason RQ3 needs a control condition (no skill at all) alongside the 4 restyling conditions, not just original-vs-restyled comparisons. Cites Ling et al. directly as [29].
→ full notes: literature_full.md#liu2026-skillsvote

### xing2026 — Recipes for Agents: Understanding Skills and Their Open Questions

Frames agent skills as modular procedural-knowledge packages injected at inference time (a "recipe" analogy), and diagnoses three structural bottlenecks limiting current agents: diminishing returns from scaling, context inefficiency (wasted token budget on redundant/failed attempts), and a reasoning-execution gap under procedural pressure. Defines a skill as scope declaration + procedural instructions + failure/fallback conditions, and lays out 10 open research questions (runtime selection, construction scope/composition, evaluation/trust, portability, governance, training-data implications). Purely conceptual/positional — no linguistic or stylistic analysis, no cross-ecosystem clustering, no execution-outcome testing. Orthogonal to RQ1/RQ2/RQ3; useful only as a definitional/governance citation.

### ai2026 — SciVisAgentSkills: Design and Evaluation of Agent Skills for Scientific Data Analysis and Visualization

Introduces domain-specific procedural skills for scientific visualization (ParaView, molecular dynamics/VMD, topology/TTK, object identification) plus SciVisAgentBench, a 100-task-step benchmark across five domains. Skills improve mean task scores for Codex/Claude Code agents, with gains varying by task type and harness verbosity (Claude Code costs more tokens than Codex for the same skill). Purely functional (with-skill vs. without-skill), not linguistic — no adoption metrics, no prose-style manipulation. **Directly useful for RQ3 pilot design:** their eval stack (pass@K, completion rate, image-quality metrics PSNR/SSIM/LPIPS, token-cost tracking, repeated trials for noise reduction) is a ready template for the restyling pilot's task suite and metrics. Also flags a confound to control for: skill benefit is harness/caching-dependent, not just content-dependent.
→ full notes: literature_full.md#ai2026

### xie2026-harness-survey — A Survey on AI Agent Harness

Systems-level survey of ~150 recent works (2023–2026) proposing a unified four-layer harness taxonomy: Execution & Orchestration (routing, model selection, multi-agent composition), Context & Trajectory Management (state compaction, memory, trajectory persistence), Interaction Surface & Execution Environment (protocols, tool use, code execution), and Constraints & Guardrails (sandboxing, permissions, injection defenses). Covers skill-acquisition design patterns (practice, self-evolution, discovery) and memory architectures, and argues for an asymmetric co-evolution between models and harnesses shifting capability toward harness-native execution/governance. Purely architectural: no linguistic/stylistic measurement of skill documentation, no adoption or execution-outcome testing. Low relevance — represents exactly the structural-analysis-only approach RQ1/RQ2/RQ3 extend beyond.

### holzbauer2026 — Context Matters: Repository-Aware Security Analysis of the Agent Skill Ecosystem

Security analysis of 238,180 skills (ClawHub, SkillDirectory, Skills.sh, GitHub). Shows scanner disagreement across platforms (14–42% "malicious" depending on scanner) and that repository-context analysis collapses false positives to 0.52%. No linguistic features, no adoption/execution measurement. Low relevance — cite at most as one more corpus source if we ever need repository-level context, otherwise skip.

### gao2026 — From Registry to Repository: How AI Agent Skills Are Written, Adapted, and Maintained

First large-scale empirical study of skill *maintenance*: 18,463 skills.sh + 23,199 GitHub personal-use skills, 3,709 recovered reuse linkages (LCS similarity ≥0.99, 70% precision), skills mapped to SWEBOK knowledge areas, and 444 hand-coded edits across 6 categories (metadata, spec, constraints, resources, compatibility, presentation). Finding: most reuse is near-verbatim; 53% of updates are additive. Categorical/behavioral analysis, not linguistic — but the **LCS-based reuse-linkage method is directly reusable for our own near-dup/update-survival detection in RQ2**, and the edit taxonomy is a useful frame for what "skill evolution" looks like structurally (complementary to, not competing with, our linguistic-cluster framing).
→ full notes: literature_full.md#gao2026

### yang2026 — FederatedSkill: Federated Learning for Agentic Skill Evolution

Privacy-preserving framework exchanging semantic skill patches (ADD/EDIT/DELETE) across clients instead of raw trajectories, with a server-side POMDP evolution agent. +44.4% success rate, -37.5% cost vs. self-evolution baselines on SkillFlow (20 task families); privacy audit shows negligible entity leakage. Execution outcomes are architecture-driven (federated patch exchange), not prose-driven — no linguistic feature analysis, no instruction-dialect clustering. Sits in a dense, mutually-citing skill-evolution sub-literature (not citing or cited by Ling et al.) that is a parallel silo to our linguistic angle. One shared benchmark worth noting: SkillFlow.
→ full notes: literature_full.md#yang2026

### qiu2026 — Unsupervised Skill Discovery for Agentic Data Analysis (DataCOPE)

Unsupervised framework discovering data-analysis skills from agent exploration trajectories via contrastive learning over verifier-scored behavioral patterns (+9.71%/+32.30% over baselines on two benchmarks). Trajectory-based discovery, not document-linguistic analysis — no relevance to RQ1/RQ2, marginal to RQ3 (connects skill quality to execution, but via a completely different mechanism). Low relevance, no further engagement needed.

### zhang2026 — SkillComposer: Learning to Evolve Agent Skills for Specification and Generalization

Three learnable skill operations — create/merge/improve — trained via rejection sampling on delta pass-rate; +4.5/+3.4 over baselines on τ²-Bench/LiveCodeBench. Validates that skill-text rewriting affects execution outcomes (supports RQ3's core premise) but rewrites content and prose together rather than isolating linguistic style — the opposite of our content-preserving-restyling design. Notably does **not** test cross-ecosystem generalization (Claude Code/OpenClaw/Hermes), which is exactly the axis RQ1 cares about. Useful as an adjacent "skill composition" citation, not a methodological precedent.
→ full notes: literature_full.md#zhang2026

### li-codeskill — CODESKILL: Learning Self-Evolving Skills for Coding Agents

RL-trained skill-management policy (extract/evolve/maintain skill banks from coding-agent trajectories) with hybrid rubric + downstream-execution rewards. +9.69 over no-skill baseline, +4.01 over prompt-based methods on EnvBench/SWE-Bench Verified/Terminal-Bench. Skill-content extraction and management-policy learning, not linguistic measurement. Useful background for RQ3's execution-benchmark methodology (which benchmarks to use, how to score) but no direct feature-extraction or adoption contribution.

### xu2026 — An Empirical Study of Agent Skills for Healthcare: Practice, Gaps, and Governance

557 healthcare skills from ClawHub, hand-annotated on 11 dimensions (function, care-cycle stage, primary user, input modality, autonomy level L1–L5, clinical-decision impact, risk level, safety-boundary strength, language, geography). Highly multilingual (65% EN/25.7% ZH), extreme contributor concentration (top 5 = 35% of skills). Human-annotated structured taxonomy, not automated linguistic features — but a strong **worked example of a stratified-corpus + adoption-proxy methodology** with control variables well beyond our planned length/topic/ecosystem set; worth stealing a few control dimensions (autonomy level, input modality) for RQ2 if we want richer covariates than the plan currently specifies.

### li-skillhone — SkillHone: A Harness for Continual Agent Skill Evolution Through Persistent Decision History

Role-separated architecture (Proposer/Explorer/Developer/Reviewer/Decider on the optimization side; Executor/Analyzer/Reporter/Auditor on a separate evaluation-only repository) with a persistent per-step decision record `h_t = (q_t, r_t, e_t, o_t)`. +14–28 pts over baselines on GAIA/WebWalkerQA; deployment study shows +18.8 pts average across seven recurring tool-mediated scenarios. No linguistic feature extraction, no instruction-dialect clustering. The optimization/evaluation role separation (to prevent evaluation-side leakage) is a good design note for RQ3's pilot harness — Timbro's dogfooding verification step should stay strictly separate from whatever generates the restyled variants.
→ full notes: literature_full.md#li-skillhone

### preprints202605-1276 — A Survey of Agent Skills: Toward Procedural Infrastructure for LLM Agents

Proposes a six-layer taxonomy of agent skills: ontology (conceptual status, knowledge compression), representation (natural-language guidance, code snippets, decision graphs, filesystem packages, structured records), lifecycle (acquisition, storage, retrieval/composition, usage, maintenance/deprecation, internalization), runtime integration (terminal/tool interfaces, multi-agent systems, harnesses), governance (security/risk/trust), and applications (robotics, games, web, GUI/OS agents, software engineering). Surveys skill representations across Claude Code, Codex, OpenClaw, Hermes, and other ecosystems, formalizing skills as S = (C, P, R, E). Does not measure linguistic features or test execution-outcome changes under restyling. High relevance as a positioning/ecosystem-landscape reference for RQ1 — the closest thing among the previously-unread batch to a direct "instruction dialect" framing, though at a taxonomic rather than measured-linguistic level.
→ full notes: literature_full.md#preprints202605-1276

### preprints202604-1817 — A Survey of Agent Skills for Foundation-Model Agents: Concepts, Representations, Lifecycles, Evaluation, and Applications

Organizes agent skills as reusable procedural-competence units bridging foundation-model capability and task execution, formalized as σ = (Cσ, Πσ, Tσ, Iσ) (applicability conditions, internal procedure, termination criterion, callable interface). Structures the literature across five dimensions — representation (textual/programmatic/hybrid), functional scope/abstraction level, acquisition source, retrieval visibility, and lifecycle (discovery → practice/distillation → storage → composition → evaluation → update) — arguing representation is foundational since it determines reusability, transferability, and whether composition/auditing stay tractable. Taxonomic/organizing survey, not an empirical study: no linguistic feature extraction, no statistical clustering of syntax/style, no execution-outcome measurement. Medium relevance — grounds RQ1's representation-dimension framing, but contributes no measured baseline on the linguistic/execution axes our RQs target. Maintains a public curated paper list (github.com/JinhaoShen/awesome-agent-skill-papers) worth checking for adjacent unindexed work.
→ full notes: literature_full.md#preprints202604-1817

### openreview-fNXXsp9iub — They Are Not Static: A Survey of Dynamic Agent Skills

Surveys 94 papers (2023–2026, corpus frozen 2026-04-24) on dynamic agent-skill systems via a formal 7-tuple lifecycle model (evidence acquisition, proposal, verification, admission, storage, retrieval, maintenance, governance), organizing 82 representative systems into eight architectural families and a master coding sheet (operator vocabulary, infrastructure topology, pipeline architecture, verification form). Introduces five evidence-graded design patterns (e.g., admission gates drop verification failures 71%→41%; retrieval scales sublinearly under distractor load; maintenance ablations show 4.5× utilization gains as skill count grows) and catalogs eight safety surfaces (prompt injection, supply-chain poisoning, credential leakage, ownership/attribution) with quantitative anchors per lifecycle stage. Purely architectural/structural: zero linguistic feature extraction, zero execution-outcome measurement tied to prose. Low relevance to RQ1–RQ3 directly, but useful as an ecosystem-landscape reference, and its evidence-grading scheme (A/B/C/D by causal strength) is a reusable rigor template for how we grade our own RQ3 findings.
→ full notes: literature_full.md#openreview-fNXXsp9iub

## Executive summary (2026-07-06 batch)

Across all papers now read (including the 5 that were previously blocked by 403s, recovered via user-supplied PDFs), nobody extracts deterministic linguistic/syntactic features from skill prose or links prose-only restyling to execution outcomes — our gap remains open. They cluster into: (a) architecture/systems reviews (`zhou2026`, `xie2026-harness-survey`) — background citation only; (b) taxonomic/positioning surveys (`xing2026`, `preprints202605-1276`, `preprints202604-1817`, `openreview-fNXXsp9iub`) — organize the skill ecosystem/lifecycle/representation space conceptually, useful for RQ1 framing but no measured linguistic or execution data; (c) marketplace/registry infrastructure & evaluation (`liang2026-skillnet`, `li2026-agentskillos`, `hu2026-clawhub`, `liu2026-skillsvote`, `gao2026`, `xu2026`, `holzbauer2026`) — build/characterize large corpora via structural, functional, orchestration, or security metrics, never linguistic ones; (d) a dense, mutually-citing skill self-evolution sub-literature (`yang2026`, `zhang2026`, `li-codeskill`, `li-skillhone`, `qiu2026`) — rewrites skill *content* to improve execution, treats prose as a black box; (e) execution/benchmark design (`ai2026`, `li2026-agentskillos`) — reusable eval methodology for the RQ3 pilot (pass@K/multi-metric scoring, Bradley–Terry pairwise judging).

**Highest-value takeaways:** `hu2026-clawhub`'s cross-lingual clustering is the closest existing precedent for "instruction dialects" (RQ1), at a topic rather than linguistic level. `preprints202605-1276` and `preprints202604-1817` supply the most direct RQ1 conceptual scaffolding (representation/ontology dimensions across ecosystems) but stop at taxonomy, not measurement. `gao2026`'s LCS reuse-linkage method and `xu2026`'s stratified-annotation methodology are directly reusable for RQ2's dedup and covariate design. `ai2026` and `li2026-agentskillos` jointly supply reusable RQ3 pilot eval methodology (multi-metric benchmark design; bias-mitigated pairwise judging). `liu2026-skillsvote` is a reason to add a no-skill control condition to RQ3, since it shows irrelevant skill exposure can actively hurt task success, not just fail to help. `openreview-fNXXsp9iub`'s evidence-grading scheme (A/B/C/D by causal strength) is a reusable rigor template for grading our own RQ3 findings.
