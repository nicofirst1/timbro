## Paper summaries

### zhou2026 — Externalization in LLM Agents: A Unified Review of Memory, Skills, Protocols and Harness Engineering

Systems-level review framing LLM agent externalization across three dimensions — Memory, Skills, Protocols — as a Norman-style cognitive-artifact transformation. The Skills section (§4) covers specification, discovery, progressive disclosure, execution binding, and composition, plus four acquisition pathways (authored, distilled, discovered, composed). Purely architectural: no linguistic/stylistic measurement of skill documentation, no adoption or execution outcomes. Orthogonal to our contribution — treats skills as architectural objects, we treat skill files as linguistic documents with measurable features tied to adoption/execution. Good one-line intro citation for "why skills matter" framing; nothing to reuse methodologically.
→ full notes: literature_full.md#zhou2026

### liang2026-skillnet — SkillNet: Create, Evaluate, and Connect AI Skills

Proposes a three-layer skill ontology (taxonomy, relation graph, package library), an automated multi-source skill-creation pipeline, and a 5-dimension evaluation protocol (Safety, Completeness, Executability, Maintainability, Cost-awareness) validated with near-perfect human/LLM inter-rater agreement (QWK ~1.0). Curates 200K+ skills across multiple ecosystems; shows +40% reward / -30% interaction steps on ALFWorld/WebShop/ScienceWorld versus ReAct/Expel/Few-Shot baselines. Compares six marketplaces (ClawHub, SkillsMP, SkillHub, Skills.sh, SkillNet, others). No linguistic feature analysis, no prose-style variation. RQ3 gains a validated Executability-evaluation precedent; RQ1 gains nothing (content/quality, not language).
→ full notes: literature_full.md#liang2026-skillnet

### li2026-agentskillos — Organizing, Orchestrating, and Benchmarking Agent Skills at Ecosystem Scale

Shanghai AI Lab paper proposing *AgentSkillOS*: (1) **Manage Skills** — organizes a large skill ecosystem into a *capability tree* via recursive LLM-driven categorization, so retrieval can go coarse-to-fine instead of flat semantic search over the whole pool; (2) **Solve Tasks** — retrieves a relevant skill subset from the tree, composes them into a DAG (three orchestration strategies: quality-first, efficiency-first, simplicity-first), then executes the DAG with dependency-aware layering. Evaluated on a 30-task, 5-category benchmark (data computation, document creation, motion video, visual design, web interaction) using **LLM-based pairwise judging in both orderings + a Bradley–Terry model** to get bias-mitigated, fine-grained ranking scores — tested across three ecosystem scales (200/1K/200K skills). Key finding: DAG-based orchestration substantially outperforms flat skill invocation even with the *identical* oracle skill set — structured composition, not mere availability, is what unlocks skill value. Cites Ling et al. (2602.08004) for the "280,000+ skills publicly available" ecosystem-scale framing, and cites SkillNet as related organizing work.

Relevance to us: no linguistic feature analysis, no adoption-prediction, no RQ1 overlap. But their **Bradley–Terry pairwise-judging protocol is a strong candidate evaluation method for the RQ3 pilot** if we want continuous quality/success scores rather than binary pass/fail — position-bias mitigation (judging both orderings) is a detail worth adopting directly. Their scale experiment (200 → 200K skills) is also a useful precedent for how retrieval/orchestration degrades as a skill library grows, tangential to but consistent with our corpus-scale concerns.
→ full notes: literature_full.md#li2026-agentskillos

### hu2026-clawhub — Red Skills or Blue Skills? A Dive Into Skills Published on ClawHub

Analyzes 26,502 ClawHub skills via a three-stage crawl+normalize pipeline: functional clustering (TF-IDF + k-means, 10 clusters/language), download-distribution analysis (log-normal, most skills 30–60 downloads, <10% exceed 1,000), and risk classification (12 classifiers, best Logistic Regression 72.62% acc / 78.95% AUROC, documentation completeness as top feature). Key finding for us: **English skills cluster around infrastructure/tooling (Developer Tools 36.3%), Chinese skills around application scenarios (media, social, finance)** — the closest existing empirical hint that ecosystems/languages produce distinct "dialects," though measured via bag-of-words topic clustering, not linguistic features. Their download distribution is a useful adoption-proxy baseline for RQ2. Corpus/crawl methodology (identity/metadata/content/risk/effectiveness normalization) is transferable to our own corpus build.
→ full notes: literature_full.md#hu2026-clawhub

### liu2026-skillsvote — SkillsVote: Lifecycle Governance of Agent Skills from Collection, Recommendation to Evolution

MemTensor paper treating agent skills as *lifecycle artifacts*: profiles a **million-scale open-source skill corpus** for environment requirements, quality, and verifiability; before execution, runs agentic library search (not static semantic matching) over structured skill folders to expose only relevant instructional context; after execution, decomposes trajectories into skill-linked **subtasks** (the smallest unit with one objective, one evaluation signal, at most one associated skill) and attributes outcomes to skill-guided execution vs. independent exploration vs. environment; only successful, reusable subtask evidence is allowed to trigger an "evidence-gated" skill-library update (edit existing skill / create new skill / skip). Evaluated on Terminal-Bench 2.0 and SWE-Bench Pro across three Codex model/effort configs (GPT-5.2 medium, GPT-5.4 mini medium, GPT-5.5 xhigh): improves avg@5 accuracy on Terminal-Bench 2.0 by +0.9–2.6pp and avg@1 resolve rate on SWE-Bench Pro by +1.2–2.7pp over no-skills baselines, beating ReasoningBank and a naive skill-creator baseline on most settings. Also shows pre-task recommendation reduces *negative* transfer from irrelevant/weakly-related skills (mean task-level contribution goes from +3.3/-6.7 without recommendation to a balanced +6.0/-6.0 with it).

### cho2026 — A Concise Agent is Less Expert: Revealing Side Effects of Using Style Features on Conversational Agents

Surveys 127 ACL Anthology papers to identify 12 commonly-used conversational-agent style features (friendly, helpful, concise, etc.), then runs controlled synthetic dialogues (task-oriented + open-domain) where an LLM is *prompted* to adopt one style feature at a time and a separate LLM-judge (pairwise) rates how that steering affects perception of the other 11 features. Introduces the CASSE dataset and tests activation-steering as a mitigation. Headline finding: prompting for conciseness significantly and unintentionally reduces perceived expertise — style features are "deeply entangled rather than orthogonal," not independently controllable knobs. Everything is subjective LLM-judged perception on conversational chat replies; there is no measurement of linguistic features directly from text (style is a prompted label, not an extracted one) and no real task-execution/success outcome.

This is our **second named novelty competitor** (README §2), and the differentiation is the sharpest one in the whole related-work section: (i) Cho manipulates style via prompting/steering and evaluates the resulting *label*, we *measure* deterministic linguistic features (imperative density, hedging, syntactic depth, POS/dep distributions) directly from existing prose; (ii) Cho's outcome is subjective LLM-judged perception, ours is adoption proxies (RQ2) and real execution success (RQ3); (iii) Cho studies conversational chat replies, we study agent-skill instruction documents. Cho's finding that steering for one style trait shifts perception of others is also the direct source of our design rule that raw length must be a covariate everywhere in RQ1–RQ3 — if unmeasured correlated style traits can flip perceived competence, an unmeasured length confound could just as easily flip an apparent adoption or execution effect.

### jeoung2026-promptprism — PromptPrism: A Linguistically-Inspired Taxonomy for Prompts

Proposes a three-level linguistically-inspired taxonomy for prompts — functional structure (role of each prompt component), semantic component (meaning/content relationships), syntactic pattern (structural/grammatical arrangement) — and demonstrates it through three applications: automated prompt refinement, dataset profiling (extracting structural/semantic/syntactic characteristics from prompt collections), and sensitivity analysis (how semantic reordering or delimiter changes affect LLM performance). It is a taxonomy, not a feature-extraction toolkit or a linguistic-feature dataset.

Relevance to us: this is the closest named work to formally treating "syntactic pattern" as a taxonomic dimension of prompts, so it is worth citing — but the full text (read via the arXiv HTML rendering) resolves the overlap question as **confirmed non-overlap**. PromptPrism's "syntactic pattern" level is *structural/formatting* markup, not linguistic parse features: component indexing/spans, directive prefix markers (`#`, `//`, `>`, numbered lists, bullets), suffix markers (colons, sentence terminators, semicolons), delimiters (newlines, tab, whitespace), and model-specific special tokens. No adoption or execution-outcome measurement; its evaluations are ROUGE-L on refinement and ANOVA on reordering/delimiter sensitivity, i.e. LLM-response-quality proxies, not real task-execution or adoption outcomes. Differentiate as taxonomic-and-qualitative vs. our numeric-and-outcome-linked.

### zhang2025-promptdatasets — Large Language Model Prompt Datasets: An In-depth Analysis and Insights

Compiles 129 heterogeneous LLM prompt datasets (>1.22TB, >673M instances) into a taxonomy and runs multi-level linguistic analysis (lexical, syntactic, semantic) across 7 representative corpora. Our contribution differs on both the corpus (agent-skill instruction files vs. general prompts) and the outcome (adoption/execution vs. filtering/classification/quality-AUC) — cite this paper as the precedent that saturates the "static quality prediction" question, freeing RQ1–RQ3 to ask about adoption and execution instead.

### reif2021, bohr2025, letruong2025, wang2025-styleimitation — style prompting cluster

reif2021 is the origin point of the whole "prompt an LLM to adopt a style" paradigm (zero-shot "augmented" style-transfer instructions, human/qualitative eval) — the same bucket as `cho2026`, cited only as ancestry. The other three are more interesting because each holds *some* form of content fixed and measures a *real* (not LLM-judged) outcome after a style/persona manipulation: bohr2025 measures deterministic code-style metrics (verbosity, doc patterns) across a fixed coding task under different system-prompt styling; letruong2025 restyles benchmark question *prompts* via persona injection and shows measured model accuracy shifts from surface style alone; wang2025-styleimitation evaluates style-imitation fidelity with automated classifiers rather than an LLM judge. None studies instruction/skill documents, adoption, or agent task-execution success — bohr2025 and letruong2025 are the closest partial precedents for RQ3's "style change → real measured outcome change" logic, but on code-chat turns and eval-question prompts respectively, not on procedural skill files an agent reads before acting.

### wegmann2021-stel, riverasoto2021-luar, wang2023-authorrepr, john2019-styletransfer, patel2024-styledistance, qiu2025-mstyledistance, patel2023-lisa, terreau2024-vibstyle — style representation learning cluster

A distinct NLP sub-literature that learns *vector representations* of style rather than reporting named linguistic features directly. STEL (wegmann2021-stel) and LUAR (riverasoto2021-luar) are the field's reference benchmarks/baselines for learned style/authorship embeddings, evaluated by similarity or cross-domain transfer, not interpretability. wang2023-authorrepr probes whether such opaque embeddings actually track style (not topic) via post-hoc interpretable-feature probing — which is itself an argument for skipping the black box: if you need interpretable probes to check what a learned embedding captured, you may as well report the interpretable features directly, which is what our `syn_*`/`posdep_*`/`lex_*`/`dict_*` groups already do. john2019-styletransfer is the canonical VAE-based latent style/content disentanglement baseline for style-transfer generation. patel2024-styledistance and its multilingual extension qiu2025-mstyledistance train embeddings over dozens of *named* style dimensions via synthetic parallel data but still collapse them into one opaque vector rather than exposing per-dimension values — the closest "named-dimension" analog to our approach in this cluster, but still not what we do. patel2023-lisa (LISA) and terreau2024-vibstyle explicitly target *interpretable* style embeddings (human-nameable dimensions, VIB-aligned axes) — the two papers in this cluster that share our interpretability goal most directly — but both still produce a learned representation rather than reporting deterministic, directly-computed feature values, and both target authorship attribution rather than instruction-document adoption or execution.

Collectively: no paper in this cluster reports deterministic per-feature values the way we do (imperative density, hedging, syntactic depth, POS/dependency distributions, readability, computed via spaCy/regex/counting with no learned component at all), and none touches agent-skill files, adoption, or execution. This cluster is best used as a "closest adjacent paradigm we deliberately avoid" contrast: where this literature learns a representation and hopes it captures style, we skip the learning step and report the features directly — trading potential representational richness for full interpretability and zero training-data dependency.

### jiang2026-sokskills, luyen2025-escoskill, tziafas2024-lrll — skill model/taxonomy cluster (outside the LLM-agent-skill bubble)

jiang2026-sokskills (SoK: Agentic Skills) is the strongest addition of this batch to RQ1's positioning: its representation × scope taxonomy explicitly names natural-language-only skill prose as a first-class representation category, distinct from code, policy, or hybrid representations — the closest any paper in the tracker comes to isolating "skills written in natural language" as a thing worth studying on its own, though it still stops at taxonomy with no linguistic measurement. luyen2025-escoskill supplies a genuinely different conceptual anchor: it bridges LLM-decomposed "skills" to the ESCO vocational/competency taxonomy (a human-labor ontology), which is a citable precedent for framing a SKILL.md file as a decomposable competency description rather than purely a software artifact — useful if the paper wants to gesture at "skill" as a construct with a life outside software engineering. tziafas2024-lrll is a robotics/RL precedent (predating the current LLM-agent-skill wave) for growing a library of *natural-language-described* reusable skills via a "skill abstractor" — an independent lineage supporting the claim that "skill = language-described procedural unit" isn't a Claude-Code-specific invention. None of the three measures linguistic features or adoption/execution outcomes; all three are positioning/framing citations for RQ1's introduction, not methodological precedents.

### clareval2026, ambigswe2025, askassume2026, clariti2026, questbench2025 — prompt/instruction underspecification cluster

This cluster studies a genuinely different textual axis than ours — instruction *completeness* (missing goals, premises, or requirements) rather than *style* — but every paper in it reinforces RQ3's core causal premise from an adjacent angle: a property of the instruction text, independent of the model's raw capability, measurably moves real agent task-success. clareval2026 (ClarEval) and ambigswe2025 (Ambig-SWE) are the strongest of the five: both construct underspecified variants of real coding benchmarks and show interactive clarification recovers substantial success (Ambig-SWE: up to +74%), directly demonstrating that a textual-completeness manipulation — held everything else fixed — changes execution outcomes on real tasks. askassume2026 and clariti2026 are follow-on/adjacent work that instead manipulates agent clarification *policy* (when to ask vs. assume, which missing-info categories matter most) rather than the instruction text itself. questbench2025 is the most tangential: it shows reasoning competence doesn't transfer to info-acquisition in math/logic/planning tasks, with no agentic tool-use or document-following setting.

None of the five studies procedural skill/instruction documents (SKILL.md-style) specifically — all operate on single-turn user requests or SWE-bench issues — which is itself worth noting as a genuine gap: **no confirmed paper tests underspecification, let alone style, as a manipulated property of an agent-skill instruction file.** This cluster is best cited in the paper as a "different independent variable, same causal logic" precedent: if instruction *completeness* already has a demonstrated causal link to agent task success (this cluster), it strengthens rather than weakens the plausibility that instruction *style* (RQ3) does too, while also making clear RQ3 must control for completeness/content as a confound, not just treat length as the only covariate.
